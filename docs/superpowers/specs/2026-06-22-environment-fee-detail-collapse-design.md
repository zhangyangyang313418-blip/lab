# 环境测试大纲费用明细折叠设计

## 目标

`/environment-outline` 页面中的 DV、PV 费用明细默认隐藏，用户需要时可分别展开查看，减少长表格对流程图阅读的干扰。

## 交互设计

- 每个阶段保留独立的费用明细标题栏，显示“费用明细”、阶段费用细则标题及现有项目摘要。
- 标题栏提供“展开费用明细”按钮，初始状态为收起。
- 点击后仅展开当前阶段的原费用明细表，按钮文字变为“收起费用明细”。
- 再次点击隐藏当前阶段表格；DV 与 PV 的展开状态互不影响。
- 按钮使用 `aria-expanded` 表达当前状态，并通过 `aria-controls` 关联对应表格内容。

## 状态与数据边界

- 展开状态仅保存在组件会话状态中，刷新页面后恢复默认收起。
- 收起时不挂载费用表格，避免隐藏内容继续占用 DOM 和无障碍阅读顺序。
- 不修改费用计算、单项费用编辑弹窗、顶部费用汇总、Excel 导出、本地草稿或模板迁移版本。

## 样式

- 沿用现有深蓝、浅灰、白色商务风格。
- 收起状态保留紧凑卡片标题和摘要；按钮具有明确的悬停、键盘焦点与展开状态反馈。
- 展开后继续使用现有横向滚动表格，不改变列宽和费用数据样式。

## 测试与验收

- 初次进入页面时，DV/PV 费用表头与明细数据均不显示。
- DV、PV 均存在独立展开按钮，且初始 `aria-expanded="false"`。
- 点击 DV 按钮后只显示 DV 表格，PV 仍保持收起。
- 再次点击 DV 按钮后表格隐藏，状态恢复为收起。
- 现有费用按钮、费用计算与页面构建测试继续通过。
