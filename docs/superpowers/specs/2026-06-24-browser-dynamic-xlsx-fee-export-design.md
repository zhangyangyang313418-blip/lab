# 网页动态 XLSX 费用导出设计

日期：2026-06-24

## 1. 目标

将 `/environment-outline` 的费用导出从无样式 SpreadsheetML `.xls` 切换为真正的 Office Open XML `.xlsx`。

网页必须根据当前平台选择对应母版：

- MLA：`outputs/mla-fee-export-template/MLA费用导出模板.xlsx`
- EMA：`outputs/ema-fee-export-template/EMA费用导出模板.xlsx`

导出时写入当前页面的动态数据，并保留模板原有的 8 个 Sheet、样式、公式、合并单元格、列宽、行高、隐藏区、原生图表和 WPS 可编辑性。

必须支持用户在环境大纲中：

- 新增测试项目
- 删除测试项目
- 新增整个 Group
- 删除整个 Group
- 修改测试项目、样品范围、测试时间、费用和特殊备注

## 2. 当前问题

页面按钮当前调用 `downloadMlaEnvironmentFeeWorkbook()`，该函数通过字符串拼接重新生成 SpreadsheetML：

- 文件扩展名为 `.xls`
- MIME 为 `application/vnd.ms-excel`
- 不读取 MLA 或 EMA 的 `.xlsx` 母版
- 只能复用数据逻辑，不能保留母版的原生 Excel 样式和图表对象

仓库内虽然已有 MLA、EMA 正式 `.xlsx` 模板，但当前只由离线脚本复制、校验，没有接入网页下载链路。

## 3. 方案选择

采用浏览器端 OOXML ZIP 层动态填充。

工作簿仍以用户确认的原始 `.xlsx` 为母版。浏览器加载母版二进制后解压 ZIP，只修改需要动态变化的 XML 部件，再重新压缩下载。

不采用 ExcelJS、SheetJS 等“读取后整体重新保存”的方案，因为这类方案可能丢失或重写：

- 原生图表
- WPS 专有兼容信息
- 手工样式
- 绘图关系
- 隐藏辅助区
- 页面设置

不新增后端生成服务。当前项目应继续保持静态前端部署能力。

## 4. 总体架构

导出链路分为四层：

### 4.1 业务数据模型层

继续使用 `mlaEnvironmentFeeExport.ts` 中已经验证的业务逻辑生成中间工作簿模型：

- 项目概要
- 样品及辅助设备需求
- 费用预估
- SGS / 华测 / 苏勃
- 费用对比
- 特殊项目费用
- 费用规则校验

该层不关心 Excel 样式，只输出每个 Sheet 的语义行、单元格值和公式信息。

### 4.2 模板选择与加载层

根据 `projectSetup.platform` 选择模板：

- `MLA`、`MLA-RHD` 使用 MLA 模板
- `EMA`、`EMA-RHD` 使用 EMA 模板

模板作为 Vite 静态资源随网页发布，使用稳定 URL 加载。生产构建必须包含两份模板。

模板加载失败时：

- 显示明确错误
- 不触发下载
- 不回退到旧 `.xls`

### 4.3 OOXML 动态填充层

新增独立模块负责：

- 解压 `.xlsx`
- 读取 workbook 与 relationship 映射
- 按 Sheet 名定位 worksheet XML
- 识别模板中的元数据区、标题行、数据行、Group 标题行、汇总行和辅助数据区
- 根据动态模型复制模板行
- 写入值和公式
- 删除旧数据区中多余行
- 重编号行号和单元格地址
- 平移受影响的合并单元格、超链接、条件格式、筛选、表格、绘图锚点和公式引用
- 更新 worksheet `dimension`
- 更新工作簿计算标记，使 WPS/Excel 打开时重新计算

共享字符串处理原则：

- 动态写入内容优先使用 `inlineStr`，避免大规模重建 `sharedStrings.xml`
- 母版原有 shared string 引用原样保留
- 数字保持数字单元格，不写成文本
- 公式单元格保留 `<f>`，缓存值可清空并交由 WPS/Excel 重算

### 4.4 下载层

重新压缩为真正的 `.xlsx` Blob：

- MIME：`application/vnd.openxmlformats-officedocument.spreadsheetml.sheet`
- MLA：`MLA测试项目及费用预估.xlsx`
- EMA：`EMA测试项目及费用预估.xlsx`

## 5. 动态区块设计

不依赖固定行号。每个 Sheet 使用语义标记和模板样式角色识别区块。

### 5.1 项目概要

项目概要区固定保留母版结构，只替换对应值：

- OEM
- 平台
- 项目名称
- 驾驶方向
- 项目类型
- 导出范围
- 是否完全复用
- 环境模板

### 5.2 样品及辅助设备需求

母版保留顶部样式和 DV/PV 布局。

根据当前环境大纲动态生成：

- Phase 标题
- Group Max
- 备样
- Phase Total
- 需求细则
- Group 内测试项目明细

新增或删除 Group 时，复制对应 Phase 中的 Group 标题、表头、数据行和 Max 行样式。公式范围必须按新行号重写。

### 5.3 费用预估

按实际 Phase 与 Group 顺序生成：

- Phase 标题行
- Group 标题行
- 表头行
- 测试项目数据行
- Group 合计行
- 附加费用区块

数据行样式取母版同类型普通行；Group 最后一行、内部费用行、委外费用行和特殊备注行分别复用最接近的模板样式角色。

汇总公式范围必须从该 Group 第一条数据行到最后一条数据行，不允许固定引用旧母版行号。

### 5.4 实验室 Sheet

SGS、华测、苏勃只保留该实验室可报价且属于委外的普通测试行，并动态生成：

- Phase / Group 区块
- 实验室费用
- 公式说明
- Group 汇总
- DV/PV 附加费用

每个实验室 Sheet 独立计算行数，不能假设三张表行数一致。

### 5.5 费用对比

顶部对比表和原生图表保留。

动态更新：

- DV / PV / DV+PV 预估费用
- SGS / 华测 / 苏勃报价
- 测试项目一致性检查
- 隐藏图表辅助区

当前图表使用固定的三阶段、四系列布局，因此分类点数量保持 14 个，不随 Group 数量变化。动态变化的是每个阶段的汇总值和缓存。

必须同步更新：

- worksheet 辅助单元格
- `chart1.xml` 中系列缓存值
- 图表引用公式

不得把原生图表替换为图片。

### 5.6 特殊项目费用

按当前特殊项目和附加费用动态生成，保留计算公式列和 Phase / Group 样式。

### 5.7 费用规则校验

动态输出所有实验室规则校验行。

MLA 的该 Sheet 保持隐藏；EMA 应统一为隐藏，避免两个模板行为漂移。

## 6. 行复制与引用更新规则

行操作必须作为同一个转换过程执行，不能先随意插入再补引用。

每次重建动态区时：

1. 保存静态顶部与静态底部区块。
2. 从母版提取各类样式原型行。
3. 根据业务模型生成新的完整动态行序列。
4. 为新行分配连续行号。
5. 重写所有单元格地址。
6. 重写新行内部公式。
7. 平移动态区之后的原有公式和引用。
8. 重建受影响的合并区域。
9. 更新表格、筛选、条件格式、绘图锚点和 Sheet 范围。

禁止：

- 只改 `<row r>` 而不改单元格地址
- 只扩展 `dimension` 而不改公式
- 删除行后留下旧合并区域
- 用硬编码数量覆盖公式

## 7. 模板资产管理

两份模板通过构建时复制进入静态资源目录，源文件仍保留在现有正式输出目录。

需要增加模板资产一致性测试：

- 静态资源与正式模板 SHA-256 一致
- 两份模板均可解压
- Sheet 顺序固定为 8 个
- 原生图表部件存在
- MLA、EMA 不得加载错模板

模板更新流程：

1. 用户确认新的正式模板。
2. 更新 `outputs/*-fee-export-template/` 中的正式文件。
3. 执行模板资产同步脚本。
4. 运行一致性和导出测试。

## 8. 错误处理

以下情况必须阻止下载并显示错误：

- 模板 HTTP 加载失败
- 模板不是有效 ZIP
- 缺少任一必需 Sheet
- 缺少模板样式原型
- 原生图表或 drawing relationship 缺失
- 动态生成后出现非法引用
- 重新压缩失败

错误信息需要指出平台与失败阶段，例如：

```text
MLA 费用模板加载失败，请刷新页面后重试。
EMA 费用模板缺少“费用对比”Sheet，无法导出。
```

## 9. 兼容性

目标环境：

- WPS Office for macOS
- Microsoft Excel
- Chrome / Codex 内置 Chromium
- Vite 开发与生产构建

输出工作簿必须保持可编辑，不启用工作表保护。

## 10. 测试策略

### 10.1 单元测试

- 平台选择正确模板
- 文件名为 `.xlsx`
- MIME 正确
- 模板加载失败不会回退 `.xls`
- 单元格值编码正确
- 数字与公式类型正确
- 行复制后地址连续
- 合并区域随行数调整
- 公式范围随新增/删除项目调整
- 图表辅助值与缓存同步

### 10.2 动态结构测试

至少覆盖：

- 基准 MLA
- 基准 EMA
- 新增一个普通测试项目
- 删除一个普通测试项目
- 新增一个完整 Group
- 删除一个完整 Group
- 同时增删多个 Group 和测试项目
- 空 Group 的防御性处理

### 10.3 工作簿结构测试

导出的文件必须：

- 以 ZIP `PK` 开头
- 包含 8 个 Sheet，名称和顺序正确
- 包含 `xl/styles.xml`
- 包含 `xl/charts/chart1.xml`
- 包含 drawing 与 relationship
- 包含原母版的列宽、行高和合并单元格
- 无 `#REF!`、`#VALUE!`、`#DIV/0!`、`#N/A`
- 汇总公式覆盖全部动态明细

### 10.4 页面集成测试

- 点击费用导出按钮只下载一次
- MLA 下载 MLA 模板结果
- EMA 下载 EMA 模板结果
- 下载扩展名为 `.xlsx`
- 加载失败时显示错误，不产生 `.xls`

### 10.5 人工验收

分别导出 MLA、EMA，并在 WPS 中检查：

- 8 个 Sheet
- 样式与母版一致
- 新增/删除项目和 Group 正确反映
- 公式可计算
- 图表可编辑且显示正常
- 隐藏校验 Sheet 状态正确

## 11. 非目标

本轮不修改：

- MLA 或 EMA 费用业务规则
- 环境大纲默认 Group
- 单价、计费基数或中值算法
- 模板视觉设计
- 费用对比图表的展示方案
- 其他材料、EMC 或总汇总 Excel 导出

## 12. 完成标准

只有同时满足以下条件才算完成：

1. 网页不再生成费用 `.xls`。
2. MLA 与 EMA 分别基于自己的正式 `.xlsx` 母版。
3. 用户动态增删 Group 或项目后，导出结构正确。
4. 样式、公式、图表和 8 个 Sheet 保留。
5. 自动测试、构建、ZIP 完整性和公式错误扫描通过。
6. MLA、EMA 文件均通过 WPS 实际打开检查。
