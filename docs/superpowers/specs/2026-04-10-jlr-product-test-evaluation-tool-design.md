# 产品测试评估工具（JLR 首版）设计文档

## 1. 目标

构建一个基于 React + TypeScript 的前端网页应用，名称为“产品测试评估工具”。首版仅支持 JLR 场景，围绕 HUD 新项目立项评估流程，帮助用户基于平台模板和本地配置数据，快速生成并调整以下输出物：

- 环境测试大纲
- 材料测试大纲
- EMC 测试项目清单
- 费用汇总 Excel

首版不接后端，所有配置与项目数据保存在浏览器本地，并支持本地导入 Excel 配置和导出 Excel / PDF。

## 2. 业务范围

### 2.1 首版支持场景

- OEM 固定为 JLR
- 项目类型固定支持“新项目”
- 平台支持 `EMA` 和 `MLA`
- 用户输入项目名，例如 `L463`
- 用户可勾选“是否复用同平台完整大纲”
- 系统先载入平台完整基线数据，再由人工选择保留、删减、增加或调整测试项
- 只有用户确认方案后，才开放导出

### 2.2 输出物定义

1. 环境测试大纲
   - 结构尽量保持与现有平台模板一致
   - `EMA` 使用 `EMA环境可靠性测试大纲.xlsx`
   - `MLA` 使用 `MLA环境可靠性测试大纲.xlsx`
2. 材料测试大纲
   - 结构尽量保持与 `材料测试大纲.xlsx` 一致
3. EMC 测试项目
   - 首版输出 Excel 清单
   - PDF 仅提供页面打印/导出基础版，正式文档可人工再编写
4. 费用汇总 Excel
   - 按不同 sheet 输出环境、材料、EMC 与总汇总

## 3. 用户流程

### 3.1 首页

用户在首页完成项目初始化：

- 选择 OEM：默认并仅支持 `JLR`
- 选择项目类型：默认并仅支持 `新项目`
- 选择平台：`EMA` / `MLA`
- 输入项目名称：例如 `L463`
- 勾选是否复用同平台完整大纲
- 点击“开始评估”

### 3.2 项目输入页

用户补充项目需求：

- 输入自然语言需求描述
- 勾选变更点或项目关注项
- 查看系统识别到的关键字标签
- 点击“生成初始方案”

### 3.3 结果页

系统展示四个结果区：

- 环境测试大纲
- 材料测试大纲
- EMC 测试项目
- 费用汇总

每个结果区均支持：

- 启用/禁用测试项
- 修改样品数量
- 修改周期
- 修改单价
- 手动新增测试项
- 删除测试项
- 实时刷新汇总结果

用户点击“确认方案”后，导出按钮启用。

### 3.4 配置数据页

用户管理本地配置：

- 导入平台模板 Excel
- 导入测试主数据 Excel
- 导入报价/周期 Excel
- 查看当前已加载配置
- 重置为示例数据
- 将配置保存到浏览器本地存储

## 5. 数据来源与映射

### 5.1 平台环境模板

来源文件：

- `资料/EMA环境可靠性测试大纲.xlsx`
- `资料/MLA环境可靠性测试大纲.xlsx`

用途：

- 作为平台完整基线数据
- 结果页默认先完整展示
- 导出环境测试大纲时按此结构生成

### 5.2 材料测试模板

来源文件：

- `资料/材料测试大纲.xlsx`

用途：

- 作为材料测试主模板
- 读取测试项结构、标准、要求、周期、样品数等字段
- 导出时保持主表与 DVP 分表结构一致

### 5.3 变更矩阵

来源文件：

- `资料/JLR设计变更测试矩阵.xls`

用途：

- 将变更点映射为 DV 和 EMC 推荐测试项
- 首版用于“推荐与预填”，不直接替代人工确认

### 5.4 费用与实验室数据

来源文件：

- `资料/JLR实验室定点及报价说明.xls`
- `资料/JLR 材料测试能力及单价报价--实验室回复汇总3.31.xls`

用途：

- 作为环境和材料费用主数据
- 提供收费单位、单价、能力、计算系数、样品数参考

## 6. 数据模型设计

### 6.1 项目基础数据

- `oem`: 当前固定 `JLR`
- `projectType`: 当前固定 `new_project`
- `platform`: `EMA | MLA`
- `projectCode`: 项目代号，例如 `L463`
- `reusePlatformTemplate`: 是否复用平台完整模板
- `description`: 自然语言需求描述
- `selectedChangeIds`: 用户勾选的变更点列表

### 6.2 测试项通用模型

每条测试项均抽象为统一结构，支持三类大纲共用：

- `id`
- `domain`: `environment | material | emc`
- `code`
- `nameZh`
- `nameEn`
- `category`
- `standard`
- `procedure`
- `requirement`
- `sampleQty`
- `durationValue`
- `durationUnit`
- `unitPrice`
- `pricingUnit`
- `cost`
- `source`
- `enabled`
- `editable`
- `notes`

### 6.3 模板模型

- 平台模板：记录环境测试 sheet 结构、列定义、默认测试项
- 材料模板：记录主表和 DVP 子表映射
- EMC 模板：记录输出列结构与默认测试项分组

### 6.4 费用汇总模型

- 环境费用列表
- 材料费用列表
- EMC 费用列表
- 总费用
- 总样品数
- 总周期

## 7. 规则与计算策略

### 7.1 平台基线加载

当用户创建新项目后：

- 若勾选复用平台模板，则完整载入对应平台的环境测试基线
- 材料测试基线默认载入标准模板
- EMC 基线默认载入 JLR EMC 规则主数据

### 7.2 变更矩阵匹配

系统根据以下两类输入匹配推荐项：

- 用户勾选的变更点
- 自然语言中识别出的关键词

输出策略：

- 将推荐项标记为 `suggested`
- 若平台基线中已存在同名或同编码测试项，则不重复新增，只追加推荐原因
- 若不存在，则作为新增候选项插入

### 7.3 重复项合并

合并优先级：

1. 测试编码相同
2. 英文测试名相同
3. 标准与类别均相同且名称高度接近

合并后规则：

- 样品数取最大值
- 周期取最大值
- 单价优先使用主数据
- 推荐原因合并展示

### 7.4 样品/周期/费用计算

基础策略：

- 样品数优先取模板或主数据默认值
- 周期优先取模板中的 `Duration` 字段
- 费用按 `sampleQty * unitPrice` 或 `duration * unitPrice` 计算
- 若存在 `pricingUnit` 和 `pricingFactor`，按扩展公式计算

用户修改任意字段后：

- 行项目费用实时刷新
- 当前 domain 小计实时刷新
- 总汇总实时刷新

## 8. 导入导出设计

### 8.1 Excel 导入

首版支持用户本地导入以下类型：

- 平台环境模板
- 材料测试模板
- 费用主数据
- 变更矩阵补充表

导入后解析为 JSON 并持久化至浏览器本地。

### 8.2 Excel 导出

导出包至少包括：

- 环境测试大纲 Excel
- 材料测试大纲 Excel
- EMC 测试项目 Excel
- 费用汇总 Excel

其中费用汇总 Excel 包含多个 sheet：

- `环境费用`
- `材料费用`
- `EMC费用`
- `总汇总`

### 8.3 PDF 导出

首版策略：

- 结果页支持浏览器打印导出 PDF
- EMC 的正式 PDF 文档不做模板还原，仅做基础导出

## 9. 本地存储设计

- 项目草稿

## 11. 首版边界

首版明确不做：

- 后端接口
- 多 OEM 支持
- 自动生成完全正式版 EMC PDF
- 实验室最优选型算法
- 自动审批流

首版优先保证：

- 页面可运行
- 平台模板可加载
- 测试项可调整
- 费用样品周期可自动汇总
- Excel 导出可用

## 12. 风险与应对

### 12.1 模板复杂度高

环境和材料模板含大量 merged cells 与复杂版式。首版应优先保证：

- 页面内数据结构准确
- Excel 导出结构接近模板
- 样式还原不追求一步到位

### 12.2 主数据存在多种计费方式

报价表中同时存在按样品、按小时、按点位、按材质种类等计费方式。首版应使用统一计费模型，并保留 `pricingUnit` 与 `pricingFactor` 扩展字段。

### 12.3 自然语言输入不可完全依赖

自然语言仅作为推荐增强，结果必须支持人工确认与手动调整，不能直接自动下结论。

## 13. 实施建议

建议按以下顺序实现：

1. 建立前端工程与目录结构
2. 定义 TypeScript 类型
3. 用示例 JSON 跑通首页、输入页、结果页、配置页
4. 接入 JLR 平台模板与报价示例数据
5. 实现匹配、合并、计算逻辑
6. 实现 Excel 导出
7. 最后补 PDF 导出
