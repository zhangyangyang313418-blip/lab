# MLA 样品及辅助设备需求 Excel 设计

日期：2026-06-22

## 目标

在现有费用预估工作簿第一页新增 `样品及辅助设备需求`。项目管理人员使用该页准备样品、工装、线束、Sensor 周边及特殊组件。

本轮先基于用户模板 `/Users/clytia/Desktop/MLA测试需求_单Sheet汇总及细则.xlsx` 生成一份带原生 Excel 公式的确认版。用户确认后，再把相同规则接入网页费用导出。

## 版式

- 保留用户模板的配色、字体、边框、合并单元格、列宽、行高和冻结窗格风格。
- 同一 Sheet 横向展示两个独立区域：DV 使用 `A:U`，`V` 为空白分隔列，PV 使用 `W:AQ`。
- DV、PV 各自包含“需求汇总”和“需求细则”，不生成跨 Phase 总计。
- 实际网页导出只展示该 Phase 当前存在的 Group 和测试项目；未选 Group 不保留空白模板行。
- 确认版 Excel 使用 MLA 全量模板数据展示公式覆盖情况。

## 计算层级

1. 测试项目行根据样品数量生成工装、投图板、HUD 周边和 Sensor 周边需求。
2. 同一 Group 内测试按顺序执行，各物料列取该组所有测试项目的最大值。
3. HUD、PCBA、投图板、线束及 Sensor 周边的 Phase 总需求为各 Group 需求之和。
4. 振动/冲击工装和防尘防水工装可跨 Group、跨时段复用，Phase 总需求取所有 Group 的最大值。
5. DV、PV 各自固定增加 3 台 HUD 备样；备样不增加任何辅助设备或耗材。
6. 数量为 0 或测试项目不存在时，不产生对应需求。

## 数量来源

- 优先使用测试行的有效样品数量或样品范围。
- 测试行未单独设置数量时，使用所属 Group 的样品数量。
- HUD 组写入 HUD 列，D-3/D-4 PCBA 组写入 PCBA 列。
- 页面样品数量改变后，所有配套数量必须按以下规则动态重算。

## 测试项目映射

### 常规投图与线束

- `L1&L4`：HUD 组的视频源板、视频源板电源线、视频源板与 PC 串口线、HUD 电源线 2m、FPD LINK 线 2m均等于样品数。
- D-3/D-4 的 `L1&L4`：上述配套同样等于 PCBA 样品数。D-4 原模板缺少 HUD/FPD 线束属于笔误，本版补齐。
- `K1/K2/K3/K4/K5/K6/K9`：常规投图板及 2m 线束等于样品数。
- `K7/K17/K20/K21/K22`：测试配套固定 1 套，不随样品数变化。
- `K23`：测试配套固定 1 套。
- `K18`：投图相关基础配套固定 1 套；HUD 电源线 2m 和 FPD LINK 线 2m 均为样品数的 2 倍，并在特殊要求中标记“线束必须为全新的”。这些线束计入数量汇总。

### 防尘防水工装

- `Particle Exposure`：防尘防水工装数量为 `MIN(样品数, 3)`；投图板及相关 2m 线束跟随工装数量。原模板 Group A 按样品数配置属于笔误。
- `K10`：防尘防水工装数量为 `MIN(样品数, 3)`，测试配套固定 1 套。
- `K13/K14`：防尘防水工装、投图板及相关 2m 线束均为 `MIN(样品数, 3)`。

### 振动/冲击工装

- 规则数量：无样品为 0；1 至 5 个样品为 3；6 个及以上样品为 6。
- `K15/K16.4`：振动/冲击工装按上述规则计算；投图板及相关配套跟随工装数量，并使用 HUD 电源线 3m 与 FPD LINK 线 3m。
- `K17`：振动/冲击工装和测试配套固定 1 套。
- D-8 HALT：该组为顺序测试，任一时段最多同时测试 3 个样品，因此 HALT 测试工装及配套按 3 套准备。

### Sensor 周边

- `K3/K4/K5/K9/K24/K26/K27/K52.351` 使用 Sensor 周边。
- HUB 数量等于样品数。
- USB 延长线、Sensor、Sensor 小板、Sensor 线均为样品数的 3 倍。

### 特殊要求

- `K22`：额外提供 3 个带 cover lens 的 top housing。
- D-3：额外提供与 PCBA 样品数相同的 PGU 模组/电机组件，并保留测试前 X-RAY 要求。
- D-4：额外提供与 PCBA 样品数相同的 PGU 模组/电机组件/开孔 bottom housing。
- E-1：样品可以为故障状态。
- E-2：样品无需满足光学性能。
- 特殊组件继续写在“特殊要求”列，不新增物料列。

## 公式设计

- 明细中的派生数量使用 Excel 公式引用同一行 HUD 或 PCBA 数量。
- Group `Max` 行使用逐列 `MAX` 公式。
- Phase 汇总中的 HUD/PCBA/耗材使用各 Group 行的 `SUM` 公式。
- Phase 汇总中的两类工装使用各 Group 行的 `MAX` 公式。
- 备样使用独立行，HUD 固定为 3，并纳入 HUD Phase 总数。
- 公式不得依赖宏、外部链接或隐藏插件。

## 平台范围

- 第一版以 MLA LHD 为规则基线。
- 规则模块应按测试编号和物料类型组织，后续可复用于 EMA 和 MLA RHD。
- 本轮确认版只验证 MLA，不提前猜测 EMA/RHD 的平台差异。

## 验收

- 打开确认版后修改任意测试行样品数量，相关配套、Group Max 和 Phase 汇总能够自动重算。
- DV 与 PV 互不引用、互不汇总。
- 备样仅增加 HUD 3 台。
- 两类工装 Phase 汇总取最大值，其余物料按 Group 合计。
- K18 线束计入数量并保留全新线束备注。
- Particle Exposure 按工装数量配置配套。
- D-4 补齐 HUD/FPD 2m 线束。
- 工作簿无公式错误，且视觉布局与用户模板一致。
